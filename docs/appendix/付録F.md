## 付録F：FAQ

### A. 基本概念に関する質問

|質問|回答|
|---|---|
|リヴァイブとは何ですか？|死亡時に記憶と感覚を過去の自分へ転送する能力です。物理的な時間移動ではなく、情報の上書き（バックアップのインストール）として処理されます|
|一般的なタイムリープとの違いは何ですか？|タイムリープは肉体ごと過去に移動する概念ですが、リヴァイブは情報のみを転送します。肉体は移動せず、過去の自分の記憶領域にデータを書き込む仕組みです|
|リヴァイブは誰が作った能力ですか？|不明です。発見されたものでも、誰かに与えられたものでもありません。特定の条件を満たした者に宿る、起源不明の現象です|
|能力を意図的に消すことはできますか？|できません。一度宿った能力を意図的に消去する方法は存在しません。能力が消失するのは、脳の過負荷による自動消失のみです|
|リヴァイブはチート能力ですか？|いいえ。回数無制限に見えますが、脳の負荷累積、情報の損傷、死亡トリガーの絶対性、セーブポイント非選択など多くの制約があり、使うほど能力者を追い詰めます|

### B. 発動・転送に関する質問

|質問|回答|
|---|---|
|生きている間に能力を使えますか？|使えません。リヴァイブは死亡時にのみオンライン状態になり、生存中は完全にオフラインです。意図的・部分的な発動は不可能です|
|転送される記憶の容量はどれくらいですか？|約250MB（圧縮状態）で、約5分間の記憶に相当します|
|記憶と感覚は同時に転送されますか？|はい。ただし、記憶フォルダと感覚フォルダは独立して管理されるため、片方だけが欠損する状態も発生しえます|
|転送された記憶はすぐに使えますか？|すぐには使えません。受信直後は認知的混乱が発生し、初期（混乱）→中期（整理）→後期（適応）の三段階を経て数時間〜数日で活用可能になります|
|転送された記憶は常に正確ですか？|いいえ。転送中に情報欠落（データの一部消失）や情報劣化（データの精度低下）が発生することがあります。特に劣化は自覚が困難で、歪んだ記憶を正しいと信じ込むリスクがあります|
|リリネルとは何ですか？|感覚データ転送時のチャネル解放量を指す概念です。痛みの情報量がリリネルを超えると、感覚が歪んだり完全に欠落したりします|
|フォルダシステムとは何ですか？|転送データを記憶と感覚に分けて格納する脳内の仕組みです。各フォルダの容量は3回分で、4回目の転送で最古のデータが上書き削除されます|
|脆弱性ウィンドウとは何ですか？|転送データの受信・展開中に発生する一時的な機能停止期間です。フリーズ（意識停止）→ブート（意識明滅）→復帰（意識回復）の三段階で進行し、特にフリーズ中は完全に無防備になります|

### C. 時間座標に関する質問

| 質問                     | 回答                                                                          |
| ---------------------- | --------------------------------------------------------------------------- |
| 戻り先を自分で選べますか？          | 選べません。戻り先は能力が自動的に選定し、能力者はそれを受け入れるしかありません                                    |
| 戻り先はどうやって決まりますか？       | 第一優先が感情的刻印（強い感情を伴った瞬間）、第二優先が物理的安全性（転送直後に死なないこと）です                           |
| どれくらい遠い過去に戻れますか？       | 死の淵で意識を保った秒数÷10が日数です。10秒なら1日前、120秒（上限）なら12日前まで戻れます                          |
| 即死した場合はどうなりますか？        | フェイルセーフが作動し、5日前の物理的に安全な場所へ自動転送されます。感情的刻印は適用されません                            |
| 120秒以上意識を保つことはできますか？   | できません。120秒は人体の生理的限界であり、能力とは無関係に決定されています。訓練や精神力で超えることは生理的に不可能です              |
| 長く意識を保つことにデメリットはありますか？ | あります。耐久時間が長いほど転送データ量が増大し、受信時の展開負荷が重くなります。「遠くに戻る」ことと「安全に戻る」ことはトレードオフの関係にあります |

### D. エラー・緊急機能に関する質問

|質問|回答|
|---|---|
|エマージェンシーコネクションとは何ですか？|脳が物理的に破壊された場合に自動起動する緊急転送機能です。「前回死んだ」という事実のみが転送され、詳細な記憶は送れません|
|ダイブエラーとは何ですか？|意識外からの死（不意打ち、睡眠中の暗殺等）により、記憶・感覚の転送が完全に失敗する現象です。時間の巻き戻りは発生しますが、情報は一切得られません|
|エマージェンシーコネクションとダイブエラーの違いは何ですか？|エマージェンシーコネクションは脳破壊時に発動し、最小限の情報（死亡した事実）は転送されます。ダイブエラーは意識外からの死で発生し、情報は一切転送されません。前者は「劣化した転送」、後者は「転送の完全な失敗」です|
|なぜダイブエラー時に時間だけ戻るのですか？|時間の巻き戻り（テンポラレル）と情報の転送（リヴァイブ）は独立したシステムです。ダイブエラー時はリヴァイブが起動しませんが、テンポラレル側は管の断絶を検知して通常通り巻き戻りを実行します|
|オブジェクトエラーとは何ですか？|記憶が過去の自分ではなく他者に送信される誤作動です。意図的に起こすことはできず、発生はごく稀です|
|サブジェクトコピーとは何ですか？|オブジェクトエラーで記憶を受信した者が、低確率でリヴァイブ能力を獲得する現象です。転送データに含まれる能力のプログラムが受信者の脳に定着することで発生すると推測されています|
|情報欠落と情報劣化の違いは何ですか？|情報欠落はデータが完全に消失する現象で、能力者は記憶の「穴」として自覚できます。情報劣化はデータが歪む現象で、能力者は歪みに気づけず正しいと信じ込みます。劣化の方がより危険です|
|感覚の空白とは何ですか？|リリネル（感覚チャネルの帯域上限）を超える痛みが転送された場合、感覚データが破損・欠落し「何も感じない」状態になることです。前回の死が想像を超えるものだったことを意味し、対策不能の最大の恐怖となります|

### E. 累積・制約に関する質問

|質問|回答|
|---|---|
|リヴァイブに回数制限はありますか？|回数そのものに制限はありません。ただし、使用するたびに脳への負荷が累積し、最終的に能力が消失します|
|脳の負荷を回復する方法はありますか？|休息を取ることで回復します。逆に、休息を取らずにループを繰り返すと記憶が古い順に消去されていきます|
|能力が消失する条件は何ですか？|脳が過負荷状態になり、記憶を一気に消去した時です。消失直前にラスト1回だけループが可能で、それを使い切ると能力は完全に消失します|
|老衰で死んだ場合はどうなりますか？|リヴァイブは発動しません。自然な死は脳が「生命の終わり」として受け入れるため、逆命令信号のトリガーになりません|
|ループの副作用にはどのようなものがありますか？|記憶の混濁（どのループの記憶か分からなくなる）、感覚の鈍化（死の痛みに慣れる）、時間感覚の喪失（主観時間と客観時間のズレ）、人格の変質（精神の摩耗）、サーカディアン撹乱（体内時計の崩壊）、脳への負荷累積があります|
|能力消失が近いことを事前に知る方法はありますか？|ありません。「次のループが最後かもしれない」という不確実性の中で判断を迫られ続けます。ただし、記憶消去が頻発し始めたら限界が近い兆候と推測はできます|
|負荷の悪循環とは何ですか？|負荷累積→記憶消去→戻り先が近くなる→短期ループ増加→さらに負荷累積、という自己強化型の悪循環です。サーカディアン撹乱による休息不能が加わると、脱出が極めて困難になります|

### F. テンポラレルに関する質問

|質問|回答|
|---|---|
|テンポラレルとは何ですか？|不変の時間の層に存在する物理法則であり、リヴァイブとは独立して存在します。物理時間の流れを維持・管理するインフラで、「時の溜まり場」を意味します|
|クラールムとオブスクールムとは何ですか？|クラールムは可変の時間が通常流れるメインパイプラインです。オブスクールムはクラールムの断絶時に時間を巻き戻すためのサブパイプライン（迂回路）です。全てのクラールムに対応するオブスクールムが存在します|
|管の破裂と漏出の違いは何ですか？|破裂は管が一本壊れる局所的な損傷で、影響範囲は限定的です。漏出は管自体が耐えた後に下流や隣接する管に圧が連鎖して複数の管が損傷するもので、影響範囲が広域的になります|
|テンポラレルの起源は何ですか？|不明です。意図的に不明としています。リヴァイブの起源が不明であるのと同様に、テンポラレルの全容を確定させないことで設定に奥行きと余白を持たせています|
|テンポラレルを人間が観測する方法はありますか？|ありません。テンポラレルは不変の時間の層に存在し、人間の認知はクラールム経由（可変の時間の層）に限定されているため、直接触れる手段は存在しません|

### G. 複数能力者に関する質問

|質問|回答|
|---|---|
|複数の能力者が同時に死んだらどうなりますか？|それぞれの能力が独立して発動します。優先順位は存在せず、互いの転送に影響しません|
|複数の能力者が同じ時間に戻ることはありますか？|基本的にはありません。戻り先は各能力者の記憶と感情的刻印に基づいて個別に選定されるため、一致することは極めて稀です|
|能力者同士で意図的に記憶を送り合えますか？|できません。記憶の転送はオブジェクトエラーという事故によってのみ発生します|
|正史とは何ですか？|能力者が「自分の経験」として採用している記憶のことです。通常はオブジェクトエラーによる転送記憶を受信する前の記憶が正史扱いですが、特定の条件下で切り替わることがあります|
|正史が切り替わると世界は変わりますか？|変わりません。正史の切り替えは能力者の主観的な認識のみに影響し、客観的な世界には一切変化を与えません|
|能力者同士が対立したらどうなりますか？|互いにループし合う消耗戦になりえます。最終的に「先に脳の限界を迎えた方が負ける」構造になり、より少ないループ回数で目的を達成できるかが勝敗を分けます|

### H. 周囲への影響に関する質問

|質問|回答|
|---|---|
|能力を持たない人はループに気づきますか？|気づきません。非能力者にとって時間は常に一方向に流れており、巻き戻りを認識する手段はありません|
|非能力者にもループの記憶は残りますか？|累積記憶として脳内に蓄積されますが、固くロックされており通常はアクセスできません。稀にロックが不完全な場合、曖昧な既視感として現れることがあります|
|なぜ非能力者にも累積記憶が残るのですか？|テンポラレルの二層構造に由来します。可変の時間が巻き戻っても、不変の時間では「一度流れた事実」が蓄積され消えません。この蓄積が非能力者の脳に累積記憶として残ります|
|ループを繰り返せば周囲の人の運命を変えられますか？|能力者が介入しない限り変わりません。非能力者は同じ状況で同じ判断を下すため、能力者が意図的に行動を変える必要があります|
|ループを客観的に観測できる存在はいますか？|いません。ループを認識できるのは能力者のみです。監視カメラ、日記、データベース等も全て巻き戻り前の状態に戻されます|

### I. 能力の発現に関する質問

|質問|回答|
|---|---|
|リヴァイブはどうやって発現しますか？|後天的発現（臨死体験、特殊な脳構造、実験、絶望）と先天的発現（出生時の臨死状態、先天的な特殊脳構造）のパターンがあります。いずれも不確実で、条件を満たしても必ず発現するとは限りません|
|走馬灯とリヴァイブは関係がありますか？|本設定では、走馬灯を「海馬による記憶の緊急圧縮処理」として再解釈しています。逆命令信号の回路が起動した痕跡である可能性がありますが、走馬灯を見たからといって能力が発現するとは限りません|
|走馬灯を見れば必ず能力者になれますか？|なれません。走馬灯（圧縮処理の起動）→回路の定着→能力の発現という複数段階があり、各段階で脱落が起こりえます。大多数の人間は何も起こらないまま通常の生活に戻ります|
|先天的能力者は自分が能力者であることを知っていますか？|知りません。先天的能力者は「最初の死」まで自分が能力者であることを知る術がなく、最初の死亡時に突然ループを経験します|

### J. 創作者向けの質問

|質問|回答|
|---|---|
|この設定を全部採用しなければいけませんか？|いいえ。必要な部分だけ取り出して使用できます。付録Dのカスタマイズ＆モジュール構成ガイドにコア／推奨／拡張／オプションの分類があります|
|設定を改変してもいいですか？|はい。作品の世界観に合わせて自由に改変してください。本資料はあくまで土台です|
|数値（250MB、3回分、120秒等）を変更してもいいですか？|はい。数値は全て作品ごとに調整可能です。変更時は付録Dの数値カスタマイズガイドで影響範囲を確認してください|
|耐久時間の公式を変えてもいいですか？|はい。「10秒＝1日」は本資料の設定であり、「5秒＝1日」や「1分＝1日」など、作品のテンポに合わせて自由に変更できます|
|エラーの種類を増やしてもいいですか？|はい。本資料で定義した3種類（エマージェンシーコネクション、ダイブエラー、オブジェクトエラー）以外に、作品独自のエラーを追加することも可能です|
|能力の起源を自作品で設定してもいいですか？|はい。本資料では起源を意図的に不明としていますが、「神が与えた」「未来からの介入」「進化の産物」など、自由に設定してください|
|複数能力者のルールを変えてもいいですか？|はい。独立発動の原則や記憶衝突のルールは本資料の提案です。作品に合わせて「先に死んだ方が優先」などの階層構造を導入することも可能です|
|リリネルのモデルを変更してもいいですか？|はい。本資料では複合型（固定上限＋状況変動）を基本としていますが、動的変動型または固定上限型に変更可能です|
|脆弱性ウィンドウの長さを変えてもいいですか？|はい。各段階の秒数は作品ごとに調整可能です。アクション重視なら短く、心理描写重視なら長くするなど、演出に合わせてください|
|テンポラレルの設定を省略・変更してもいいですか？|はい。テンポラレルを使わず「そういうものである」と処理することも、構造や名称を作品に合わせて変更することも可能です|
|走馬灯と能力の関係を作中で確定させてもいいですか？|はい。ただし、本資料では能力の起源を安易に確定させないことを推奨しています。確定事実とするか、仮説として残すか、一切言及しないかは作品のトーンに合わせて選択してください|

### K. 設計思想に関する質問

|質問|回答|
|---|---|
|設計の三原則とは何ですか？|「無限に見えて有限である」「救済に見えて呪縛である」「使うほど追い詰められる」の三つです。全ての設定要素がこの三原則を支える形で設計されています|
|なぜ回数無制限なのにチートにならないのですか？|脳の負荷累積、情報の損傷（欠落・劣化）、脆弱性ウィンドウ、サーカディアン撹乱、フォルダ容量制限、セーブポイント非選択性が組み合わさることで、使うほど能力者が不利になる構造になっています|
|なぜ能力の起源を不明にしているのですか？|起源を確定させると設定の奥行きが狭まるためです。不明のままにすることで「起源を探す物語」「能力の意志を問う物語」など多様な展開が可能になります|
|なぜ死亡がトリガーなのですか？|死亡を絶対条件にすることで「生きている間は使い放題にならない」制約を実現し、同時に「やり直すには必ず死ななければならない」という代償を課すためです|
|なぜセーブポイントを選べないのですか？|自由にセーブできると戦略性が過度に上がりチート化するためです。能力に選ばれた地点を受け入れるしかないことが、能力者の無力感と物語的緊張を生みます|
