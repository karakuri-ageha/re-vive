### 設計思想

| 項目 | 内容 |
|---|---|
| 三原則 | 1. 無限に見えて有限である<br>2. 救済に見えて呪縛である<br>3. 使うほど追い詰められる |
| 核心ルール | ・10秒耐えると1日遠くに戻れる（上限120秒）<br>・完全即死は5日前固定（フェイルセーフ）<br>・フォルダ容量は記憶・感覚各3回分 |
| 禁忌事項 | ・回数無制限をチートと誤解すること<br>・負荷システムを無視した運用<br>・能力の起源を安易に確定すること |
| 期待効果 | ・論理破綻のない死に戻り設定の構築<br>・戦略的緊張感のある物語展開<br>・能力者の孤独と苦悩の深い描写<br>・複数能力者による複雑な時系列戦争 |

### 推奨活用法

| 目的 | 推奨ルート |
|---|---|
| 能力の全体像を把握したい | 第1章→第2章→第8章の順に読む。概要、基本メカニズム、代償の構造を掴むことで全体像が理解できる |
| 自作品に導入したい | 第1章で概要を把握した後、付録D（カスタマイズ＆モジュール構成ガイド）で必要なモジュールを選定し、該当章のみを精読する |
| 自分のジャンルに合う使い方を知りたい | 付録E（ジャンル別活用ガイド）で自作品のジャンルを探し、推奨モジュール構成を確認した上で該当章を読む |
| 特定の用語や仕組みを調べたい | 付録A（用語集）で定義を確認するか、付録F（FAQ）から逆引きする。参照章の記載があるものはそこから本文へ進む |
| 数値を調整したい | 付録B（数値・定数一覧）で現在の数値を確認し、付録D（数値カスタマイズガイド）で変更時の影響範囲を把握する |
| エラーや異常系の挙動を確認したい | 付録C（エラー・異常系フローチャート／転送結果マトリクス）で全体像を掴み、詳細が必要な場合は第7章を参照する |
| 全設定を通読して整合性を検証したい | 第1章から第13章まで順に読み、その後に付録B（制約・制限一覧）で設定間の横断的な整合性を確認する |
